28 // class G4PVDivision
29 //
30 // Class description:
31 //
32 // Represents many touchable detector elements differing only in their
33 // positioning. The elements' positions are calculated by means of a simple
34 // linear formula.
35 //
36 // G4PVDivision(const G4String& pName,
37 // G4LogicalVolume* pLogical,
38 // G4LogicalVolume* pMother,
39 // const EAxis pAxis,
40 // const G4int nReplicas,
41 // const G4double width,
42 // const G4double offset=0)
43 //
44 // Division may occur along:
45 //
46 // o Cartesian axes (kXAxis,kYAxis,kZAxis)
47 //
48 // The divisions, of specified width have coordinates of
49 // form (-width*(nReplicas-1)*0.5+n*width,0,0) where n=0.. nReplicas-1
50 // for the case of kXAxis, and are unrotated.
51 //
52 // o Radial axis (cylindrical polar) (kRho)
53 //
54 // The divisions are cons/tubs sections, centred on the origin
55 // and are unrotated.
56 // They have radii of width*n+offset to width*(n+1)+offset
57 // where n=0..nReplicas-1
58 //
59 // o Phi axis (cylindrical polar) (kPhi)
60 // The divisions are `phi sections' or wedges, and of cons/tubs form
61 // They have phi of offset+n*width to offset+(n+1)*width where
62 // n=0..nReplicas-1
