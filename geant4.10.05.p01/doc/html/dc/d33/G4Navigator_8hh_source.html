29 // class G4Navigator
30 //
31 // Class description:
32 //
33 // A class for use by the tracking management, able to obtain/calculate
34 // dynamic tracking time information such as the distance to the next volume,
35 // or to find the physical volume containing a given point in the world
36 // reference system. The navigator maintains a transformation history and
37 // other information to optimise the tracking time performance.

512 // NOTES:
513 //
514 // The following methods provide detailed information when a Step has
515 // arrived at a geometrical boundary. They distinguish between the different
516 // causes that can result in the track leaving its current volume.
517 //
518 // Four cases are possible:
519 //
520 // 1) The particle has reached a boundary of a daughter of the current volume:
521 // (this could cause the relocation to enter the daughter itself
522 // or a potential granddaughter or further descendant)
523 //
524 // 2) The particle has reached a boundary of the current
525 // volume, exiting into a mother (regardless the level
526 // at which it is located in the tree):
527 //
528 // 3) The particle has reached a boundary of the current
529 // volume, exiting into a volume which is not in its
530 // parental hierarchy:
531 //
532 // 4) The particle is not on a boundary between volumes:
533 // the function returns an exception, and the caller is
534 // reccomended to compare the G4touchables associated
535 // to the preStepPoint and postStepPoint to handle this case.
536 //
537 // G4bool EnteredDaughterVolume()
538 // G4bool IsExitNormalValid()
539 // G4ThreeVector GetLocalExitNormal()
540 //
541 // The expected usefulness of these methods is to allow the caller to
542 // determine how to compute the surface normal at the volume boundary. The two
543 // possibilities are to obtain the normal from:
544 //
545 // i) the solid associated with the volume of the initial point of the Step.
546 // This is valid for cases 2 and 3.
547 // (Note that the initial point is generally the PreStepPoint of a Step).
548 // or
549 //
550 // ii) the solid of the final point, ie of the volume after the relocation.
551 // This is valid for case 1.
552 // (Note that the final point is generally the PreStepPoint of a Step).
553 //
554 // This way the caller can always get a valid normal, pointing outside
555 // the solid for which it is computed, that can be used at his own
556 // discretion.
