29 // class G4VPhysicalVolume
30 //
31 // Class description:
32 //
33 // This is an Abstract Base class for the representation of positioned volume.
34 // The volume is placed within a mother volume, relative to its coordinate
35 // system. Either a single positioned volume or many positioned volume can
36 // be represented by a particular G4VPhysicalVolume.

253 // NOTE:
254 // The type G4PVManager is introduced to encapsulate the methods used by
255 // both the master thread and worker threads to allocate memory space for
256 // the fields encapsulated by the class G4PVData. When each thread
257 // initializes the value for these fields, it refers to them using a macro
258 // definition defined below. For every G4VPhysicalVolume instance, there is
259 // a corresponding G4PVData instance. All G4PVData instances are organized
260 // by the class G4PVManager as an array.
261 // The field "int instanceID" is added to the class G4VPhysicalVolume.
262 // The value of this field in each G4VPhysicalVolume instance is the subscript
263 // of the corresponding G4PVData instance.
264 // In order to use the class G4PVManager, we add a static member in the class
265 // G4VPhysicalVolume as follows: "static G4PVManager subInstanceManager;".
266 // For the master thread, the array for G4PVData instances grows dynamically
267 // along with G4VPhysicalVolume instances are created. For each worker thread,
268 // it copies the array of G4PVData instances from the master thread.
269 // In addition, it invokes a method similiar to the constructor explicitly
270 // to achieve the partial effect for each instance in the array.
271 //
