29 // Class G4VTrackTerminator
30 //
31 // Class description:
32 //
33 // This is an interface for an object which can be called to kill a track.
34 // The type it provides is needed in case importance biasing and scoring
35 // is done at the same time.
36 // For navigation in the parallel geometry which is done by the importance
37 // biasing process (it derives from G4ParallelTransport), importance biasing
38 // has to be done before scoring (to score the correct volume).
39 // But since scoring would not be called if the importance biasing kills a
40 // track, it only tells a G4VTrackTerminator to kill the track.
41 // The scoring process implements the interface G4VTrackTerminator
42 // and is now able to kill the track after it has performed the scoring.
43
