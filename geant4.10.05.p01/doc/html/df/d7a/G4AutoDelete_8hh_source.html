30 // Class Description:
31 // This function implements a simplified "garbage collection" mechanism
32 // for G4-MT model. Objects are registered when created on the heap and they
33 // will be deleted at the end of
34 // the program (like they would be if marked as "static").
35 //
36 // Limitation:
37 // The registered object, should not
38 // contain any G4ThreadLocal data member. Note that in general,
39 // if object is to be thread-private it is unnecessary to mark
40 // any data-member as G4ThreadLocal.
41 //
42 // Performance issues:
43 // This function uses G4ThreadLocalSingleton that on its own uses
44 // locks and mutexes. Thus its use should be limited to only when
45 // really necessary.
46 //
47 // Example:
48 // class G4SharedByThreads {
49 // void calledByThreads() {
50 // G4Something* anObject = new G4Something;
51 // G4AutoDelete::Register( anObject );
52 // }
53 // };
54 //
55 // History:
56 // 28 October 2013: A. Dotti - First implementation
