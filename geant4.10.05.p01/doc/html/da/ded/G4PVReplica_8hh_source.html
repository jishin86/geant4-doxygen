29 // class G4PVReplica
30 //
31 // Class description:
32 //
33 // Represents many touchable detector elements differing only in their
34 // positioning. The elements' positions are calculated by means of a simple
35 // linear formula, and the elements completely fill the containing mother
36 // volume.
37 //
38 // G4PVReplica(const G4String& pName,
39 // G4LogicalVolume *pLogical,
40 // G4LogicalVolume *pMother,
41 // const EAxis pAxis,
42 // const G4int nReplicas,
43 // const G4double width,
44 // const G4double offset=0);
45 //
46 // Replication may occur along:
47 //
48 // o Cartesian axes (kXAxis,kYAxis,kZAxis)
49 //
50 // The replications, of specified width have coordinates of
51 // form (-width*(nReplicas-1)*0.5+n*width,0,0) where n=0.. nReplicas-1
52 // for the case of kXAxis, and are unrotated.
53 //
54 // o Radial axis (cylindrical polar) (kRho)
55 //
56 // The replications are cons/tubs sections, centred on the origin
57 // and are unrotated.
58 // They have radii of width*n+offset to width*(n+1)+offset
59 // where n=0..nReplicas-1
60 //
61 // o Phi axis (cylindrical polar) (kPhi)
62 // The replications are `phi sections' or wedges, and of cons/tubs form
63 // They have phi of offset+n*width to offset+(n+1)*width where
64 // n=0..nReplicas-1

92 // The type G4PVRManager is introduced to encapsulate the methods used by
93 // both the master thread and worker threads to allocate memory space for
94 // the fields encapsulated by the class G4ReplicaData. When each thread
95 // initializes the value for these fields, it refers to them using a macro
96 // definition defined below. For every G4PVReplica instance, there is
97 // a corresponding G4ReplicaData instance. All G4ReplicaData instances are
98 // organized by the class G4PVRManager as an array.
99 // The field "int instanceID" is added to the class G4PVReplica.
100 // The value of this field in each G4LogicalVolume instance is the subscript
101 // of the corresponding G4ReplicaData instance.
102 // In order to use the class G4PVRManager, we add a static member in the
103 // class G4LogicalVolume as follows: "static G4PVRManager subInstanceManager".
104 // For the master thread, the array for G4ReplicaData instances grows
105 // dynamically along with G4PVReplica instances arecreated.
106 // For each worker thread, it copies the array of G4ReplicaData instances
107 // from the master thread.
108 // In addition, it invokes a method similiar to the constructor explicitly
109 // to achieve the partial effect for each instance in the array.
110 //
111 typedef G4GeomSplitter<G4ReplicaData> G4PVRManager;
112 
113 // This macro changes the references to fields that are now encapsulated
114 // in the class G4ReplicaData.
115 //
116 #define G4MT_copyNo ((subInstanceManager.offset[instanceID]).fcopyNo)
